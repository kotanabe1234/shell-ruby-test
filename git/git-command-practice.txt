gitとは
  ファイルのバージョン管理が簡単にできるツール

gitはなんのために使うのか？
  ファイルのバージョンを管理するため。
  例）ファイルのバージョンを管理しなかった場合(複数人で開発)
   ・最新のファイルがどれかわからない
   ・誰がいつ、何を変更したかわからない
   ・他人のファイルに対して上書きしてしまったなど

  例）ファイルのバージョンを管理した場合(複数人で開発)
   ・ファイルの最新の状態が分かる
   ・いつ、誰が何を変更したかが分かる
   ・以前の状態にも戻せる


#変更履歴を確認することができる
  ・git log

#一行で表示
  ・git log --oneline

#表示するコミット数を制限
  ・git log -n<コミット数>

#現在の変更状況を確認
  ・git status

#追加・変更されたファイルなどをステージに追加する
  ・git add
    (まとめて追加する場合は[git add .]で追加できる)

#git addする前の変更分
  ・git diff

#git addした後の変更分
  ・git diff --staged

#リモートリポジトリへ送信
  ・git push

#リモートリポジトリからプロジェクトをコピーしてくる
  ・git clone

#リモートリポジトリから情報を取得してマージを行う
  ・git pull

#バージョン管理したくないファイルをGitの管理から外す
.gitignore
